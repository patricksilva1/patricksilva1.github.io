    • 1. Visão Geral


    Sky Aviator


    A Sky Aviator é uma agência de vendas de passagens aéreas especializada em proporcionar uma ampla seleção de passagens aéreas. A ideia surgiu a partir de uma conversa entre amigos que buscavam criar um sistema de passagens mais ágil e acessível do que os disponíveis atualmente. Desenvolvi um backend utilizando uma arquitetura de microsserviços, o que nos permite alcançar maior escalabilidade e garantir a independência entre os serviços. Dessa forma, podemos oferecer um sistema robusto e eficiente aos nossos clientes. Além disso, o banco de dados é preenchido com informações de voos já realizados anteriormente. Isso significa que, ao fazer uma pesquisa no sistema, você encontrará voos reais que foram operados. Isso nos permite oferecer uma experiência mais precisa de como uma agência de viagens de fato opera, ajudando-os a tomar decisões informadas sobre suas viagens. Abaixo você pode conferir mais detalhes sobre o projeto.

    2. Engenharia Requisitos


    Requisitos funcionais:


    Pesquisa de Passagens:


    - Permitir aos usuários pesquisar passagens aéreas com base em destinos, datas e preferências.

    - Apresentar resultados de pesquisa com opções de voos disponíveis, incluindo companhias aéreas, horários e preços.


    Reservas e Compra de Passagens:


    - Possibilitar aos usuários selecionar e reservar passagens aéreas desejadas.

    - Oferecer opções de pagamento seguras e integradas para a compra das passagens selecionadas.

    - Emitir bilhetes eletrônicos ou fornecer informações de reserva para os clientes.


    Gerenciamento de Clientes:


    - Manter um banco de dados de clientes com informações de contato e histórico de viagens.

    - Permitir que os clientes acessem suas reservas e informações de voo por meio de uma conta pessoal.

    - Enviar notificações e atualizações relevantes aos clientes sobre suas viagens.


    Suporte ao Cliente:


    - Disponibilizar canais de atendimento ao cliente para auxiliar em dúvidas, consultas e solução de problemas relacionados às passagens aéreas.

    - Fornecer suporte antes, durante e após a viagem, incluindo assistência em caso de alterações de voos, cancelamentos ou reembolsos.


    Requisitos não funcionais:


    São requisitos que expressam condições que o software deve atender ou qualidades específicas que o software deve ter. Em vez de informar o que o sistema fará, os requisitos não-funcionais colocam restrições no sistema.


    O software deve ser compatível com os browsers IE (versão 5.0 ou superior) e Firefox (1.0 ou superior);

    Usabilidade:

    - Oferecer uma interface intuitiva e fácil de usar para os usuários navegarem e reservarem passagens aéreas.

    - Realizar qualquer pesquisa de passagem entre todos os bancos em tempo inferior a 5 segundos.

    - Garantir tempos de resposta rápidos ao realizar reservas no sistema, como pesquisar voos.


    Desempenho:

    - Lidar com um grande volume de consultas e pesquisas simultâneas de passagens aéreas.

    - Tempo de carregamento rápido do sistema e processamento eficiente de dados.

    - Garantir um tempo de carregamento rápido das informações e processamento eficiente dos dados.


    Confiabilidade:

    - Manter a estabilidade e disponibilidade do sistema, minimizando falhas e interrupções durante o processo de reserva e compra de passagens.

    - Realizar backups regulares dos dados para evitar perdas de informações importantes.


    Segurança:

    - Proteger dados pessoais e financeiros dos clientes, utilizando criptografia e práticas de segurança adequadas.

    - Implementar autenticação segura e controle de acesso para as contas dos clientes.


    Escalabilidade:

    - Capacidade de dimensionar o sistema para lidar com o aumento do número de usuários e demanda por passagens aéreas.

    - Garantir que o sistema possa suportar picos de tráfego durante períodos de alta temporada ou promoções especiais.


    Compatibilidade:

    - Ser compatível com uma variedade de dispositivos, incluindo computadores, tablets e smartphones, para permitir que os usuários acessem o sistema de qualquer lugar.

    - Integrar-se a sistemas de pagamento e outras plataformas relevantes para oferecer opções de pagamento e serviços mais abrangentes.


    Manutenção:

    - Facilidade de manutenção do sistema, permitindo atualizações regulares, correções de bugs e melhorias contínuas para garantir a qualidade e aprimorar a experiência do usuário.


    3. Design do Sistema


    Abaixo, apresento um 'System Design' detalhado do projeto, com o objetivo de facilitar o entendimento e examinar minuciosamente os detalhes do sistema.





    A construção de sistemas de passagens aéreas é um processo desafiador, devido à natureza complexa e aos requisitos específicos envolvidos. Esses sistemas precisam lidar com uma variedade de tarefas complexas, como pesquisa de voos, reservas, integração com companhias aéreas, processamento de pagamentos, gerenciamento de clientes e muito mais. Uma das principais dificuldades na construção de sistemas de passagens aéreas é lidar com a enorme quantidade de dados em tempo real.


    Esses sistemas devem ser capazes de buscar e exibir informações atualizadas sobre voos, horários, disponibilidade de assentos e preços, garantindo uma experiência de usuário ágil e precisa. Além disso, a escalabilidade é um fator crucial nesse tipo de sistema, já que eles precisam lidar com um grande volume de tráfego, especialmente durante períodos de alta demanda, como feriados e férias.


    Garantir que o sistema possa se expandir e acomodar um número crescente de usuários é essencial para oferecer um serviço confiável e eficiente. Outro desafio enfrentado na construção de sistemas de passagens aéreas é a integração com diferentes companhias aéreas. Cada companhia pode ter seus próprios sistemas e protocolos de comunicação, o que requer uma estrutura flexível e adaptável para garantir a sincronização e a disponibilidade em tempo real dos dados.


    Em resumo, a construção de sistemas de passagens aéreas apresenta uma série de desafios devido à complexidade envolvida. No entanto, este sistema foi concebido com o objetivo de superar essas dificuldades. Ele foi projetado para ser facilmente escalável e independente, seguindo os padrões de um sistema comercial de passagens aéreas.

    SHA256Helper.java

    
                   import java.security.MessageDigest;

                   public class SHA256Helper {
                   
                       public static String hash(String data) {
                   
                           try {
                   
                               MessageDigest digest = MessageDigest.getInstance("SHA-256");
                               byte[] hash = digest.digest(data.getBytes("UTF-8"));
                   
                               StringBuilder hexadecimalString = new StringBuilder();
                   
                               for (int i = 0; i < hash.length; i++) {
                                   String hexadecimal = Integer.toHexString(0xff & hash[i]);
                   
                                   if (hexadecimal.length() == 1) {
                                       hexadecimalString.append('0');
                                   }
                   
                                   hexadecimalString.append(hexadecimal);
                               }
                   
                               return hexadecimalString.toString();
                   
                           } catch (Exception e) {
                               throw new RuntimeException(e);
                           }
                       }
                   }
